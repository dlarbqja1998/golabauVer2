# 탈퇴 기능 작업 내역

## 1. 추가된 동작

- 마이페이지에 `위험 구역`과 `회원 탈퇴 진행` 버튼을 추가했습니다.
- 탈퇴 시 확인 문구 `탈퇴하겠습니다`를 정확히 입력해야 서버 액션이 실행됩니다.
- 진행 중인 모임 방장 상태나 활성 신청 내역이 있으면 탈퇴를 막습니다.
- 탈퇴 성공 시 세션 쿠키를 삭제하고 홈으로 이동합니다.

## 2. 서버 처리 방식

- 사용자 계정을 하드 삭제하지 않고 `DELETED` 상태로 전환합니다.
- 이메일은 익명 값으로 치환합니다.
- 닉네임은 `탈퇴한 사용자`로 고정합니다.
- 프로필, 연락처, 학과, 학년, 성별 등 개인정보 컬럼은 `null` 처리합니다.
- 푸시 구독 정보는 탈퇴 시 함께 삭제합니다.

## 3. 인증 흐름 변경

- `getUserBySessionId`는 `DELETED` 계정을 더 이상 반환하지 않습니다.
- `hooks.server.ts`는 탈퇴 계정 세션을 발견하면 쿠키를 지우고 로그인 화면으로 보냅니다.
- 카카오 로그인 콜백은 이미 탈퇴된 계정을 만나면 바로 로그인시키지 않습니다.

## 4. 남은 작업

- 실제 DB에 탈퇴 관련 컬럼을 추가하는 마이그레이션 적용
- 탈퇴 사유 통계 수집 여부 결정
- 커뮤니티/댓글 화면에서 익명 작성자 표시를 추가로 점검
- 실제 운영 데이터 기준으로 탈퇴 제한 조건 재검증
