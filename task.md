# 골라봐유 서비스 점검 보고서

## 1. 현재 서비스 요약

이 프로젝트는 단순 맛집 리스트가 아니라 아래 기능이 합쳐진 생활형 캠퍼스 웹앱입니다.

- 카테고리별 식당 탐색
- 검색, 상세 페이지, 평점, 키워드 리뷰
- 커뮤니티형 리뷰 피드(`골라봐슈`)
- 룰렛 추천
- 밥약/과팅 매칭(`만나보슈?`)
- 포인트/상점
- PWA, 푸시 알림, PostHog 분석

기능 수는 많고 방향도 명확합니다. 다만 사용자 입장에서 "왜 써야 하는지", "어떻게 시작하면 되는지", "얼마나 믿을 수 있는지"를 설명하는 레이어가 아직 약합니다.

## 2. 사용자 입장에서 부족한 것

### 가장 먼저 느껴질 부족분

- 서비스 소개가 없음
  - 첫 방문자가 홈에 들어와도 이 서비스가 정확히 무엇인지 3초 안에 이해하기 어렵습니다.
  - 식당 추천 앱인지, 학교 생활 앱인지, 커뮤니티인지, 소개팅/밥약 앱인지 한 번에 정리되지 않습니다.

- 데이터 신뢰 설명이 없음
  - 식당 정보, 거리, 평점, 키워드, 리뷰 수가 어디서 오고 얼마나 최신인지 안내가 없습니다.
  - 사용자는 "이거 믿어도 되나?"를 바로 묻게 됩니다.

- 리뷰/평점 참여 가이드가 없음
  - 키워드 리뷰, 별점, 피드 작성이 왜 필요한지 보상이 무엇인지 잘 드러나지 않습니다.
  - 포인트 시스템도 존재하지만 규칙이 설명되지 않습니다.

- 만나보슈 기능의 안전/운영 정책이 부족함
  - 연락처 노출 시점은 안내하지만, 신고/차단/노쇼/비매너 대응 기준이 전면에 보이지 않습니다.
  - 이 기능은 신뢰가 핵심이라 정책 문구가 훨씬 중요합니다.

- 검색/목록의 판단 보조 정보가 부족함
  - 영업시간, 대표메뉴, 가격대, 최근 리뷰 여부, 추천 대상(혼밥/단체/늦은 밤 등)이 없습니다.
  - 지금은 "이름, 거리, 평점" 중심이라 실제 선택 피로가 남습니다.

### 있으면 체감이 큰 콘텐츠

- 첫 방문용 온보딩 문구
  - "학교 근처 밥집 찾기", "리뷰 남기기", "밥약 잡기" 3단 구조 안내

- 홈/상세에 추천 큐레이션
  - 오늘 혼밥용
  - 10분 이내
  - 리뷰 많은 곳
  - 처음 온 학생 추천
  - 술약속/카페/가성비 등 상황별 묶음

- 리뷰 품질 유도 문구
  - "가격대/대기시간/양/추천 메뉴 중심으로 적어달라"
  - 저품질 한줄평보다 실제 도움이 되는 리뷰를 유도

- 신뢰 배지/지표
  - 최근 30일 리뷰 수
  - 별점 참여자 수
  - 키워드 참여자 수
  - 최근 업데이트 날짜

- 운영 정보 페이지
  - 공지사항
  - 업데이트 내역
  - 문의/제안 응답 기준
  - 신고 및 제재 기준

## 3. 구조상 빠져 있는 페이지/문서

현재 구조를 보면 기능 페이지는 많지만 "기본 페이지"가 비어 있습니다.

### 반드시 추가 추천

- `서비스 소개` 페이지
- `이용 방법` 페이지
- `공지사항` 페이지
- `개인정보처리방침` 페이지
- `이용약관` 페이지
- `신고/차단/운영정책` 페이지
- `포인트 안내` 페이지
- `FAQ` 페이지

### 특히 중요한 이유

- 로그인, 연락처 수집, 푸시 알림, 모임 기능이 있어서 개인정보/운영 고지가 필요합니다.
- 지금은 `문의하기` API는 있지만, 사용자가 참고할 기준 문서가 없습니다.

## 4. 기술/운영 관점에서 바로 보이는 문제

### 1순위 문제

- `README.md`가 기본 Svelte 템플릿 그대로 남아 있음
  - 프로젝트 설명, 실행법, 환경변수, 배포 구조, 데이터 구조가 정리되지 않았습니다.

- 타입 에러가 매우 많음
  - `check-output.txt` 기준 `svelte-check found 133 errors and 5 warnings in 16 files`
  - 사용자 화면이 당장 깨진다는 뜻은 아니지만, 유지보수 리스크가 큽니다.

- 메타/공유 정보 정리가 약함
  - `src/app.html`에 기본 OG는 있지만 설명 품질이 낮고 페이지별 SEO/OG 분기가 없습니다.
  - `og:image`는 `/thumbnail.webp`를 가리키는데 실제 파일 목록에는 `thumbnail.png`만 확인됩니다.

- 보안/운영 정책이 UI에 드러나지 않음
  - 연락처, 밥약, 모임 기능이 있는데 신고/차단/신뢰점수 정책 설명이 없습니다.

### 2순위 문제

- `hooks.server.ts`에서 User-Agent 기준 차단이 과격함
  - `curl`, `python`, `postman` 등을 막고 있습니다.
  - 악성 트래픽 방어 의도는 이해되지만, 정상적인 점검/모니터링/서버 간 호출까지 막을 수 있습니다.

- `src/app.html`에 카카오 지도 SDK 키가 하드코딩되어 있음
  - 공개 키라 하더라도 환경변수로 분리하는 편이 운영상 안전합니다.

- `static/robots.txt`는 전체 허용인데 실제 검색 유입용 구조가 부족함
  - sitemap 없음
  - 페이지별 title/description 최적화 부족
  - 카테고리/상세 페이지 검색 유입 전략이 없음

- 식당 상세 정보가 의사결정에 비해 부족함
  - 전화/주소/지도는 있지만 메뉴/가격/운영시간/휴무/혼잡도 같은 정보가 없습니다.

## 5. 사용자 경험 기준 우선 과업

### P0. 바로 해야 하는 것

- 서비스 소개/정체성 문구 정리
  - 홈 첫 화면 상단에 "무슨 서비스인지" 한 줄로 정리

- 기본 운영 문서 추가
  - 개인정보처리방침
  - 이용약관
  - 신고/차단 정책
  - 포인트 안내

- `README.md` 프로젝트 문서화
  - 로컬 실행
  - 환경변수
  - 배포 방식
  - 주요 기능
  - DB/캐시 설명

- 타입 에러 정리 시작
  - 특히 홈, 상세, 골라봐슈, 룰렛, 마이페이지의 암시적 any와 nullable 오류 우선

- OG/아이콘/공유 메타 정리
  - 실제 존재하는 썸네일 파일 기준으로 정리

### P1. 다음으로 해야 하는 것

- 홈 화면 큐레이션 블록 추가
  - 거리순 추천
  - 리뷰 많은 곳
  - 혼밥 추천
  - 오늘 인기 카테고리

- 식당 상세 정보 확장
  - 대표 메뉴
  - 가격대
  - 영업시간
  - 추천 상황 태그
  - 최근 리뷰 요약

- 리뷰 작성 UX 개선
  - "무엇을 쓰면 좋은 리뷰인지" 가이드
  - 가격/양/웨이팅/재방문 의사 선택형 입력 보강

- 만나보슈 신뢰 UX 강화
  - 신고 버튼
  - 차단 기능
  - 노쇼/비매너 안내

- FAQ 페이지
- 학교 생활형 유틸 통합 큐레이션
  - 학식
  - 버스
  - 맛집
  - 모임

  - 리뷰 기여자
  - 키워드 기여자
  - 주간 활동 랭킹

## 6. 내가 보기에 가장 큰 핵심 문제 5개

1. 기능은 많은데 서비스 설명이 약함
2. 운영 정책/신뢰 문서가 없음
3. 식당 선택에 필요한 핵심 정보가 아직 부족함
4. 타입 에러가 많아 유지보수 비용이 커지고 있음
5. README/SEO/공유 메타 같은 기본 제품 완성도가 낮음

## 8. 결론

이 서비스는 "기능이 없는 상태"가 아니라, 오히려 기능이 먼저 많이 올라간 상태입니다.
지금 필요한 건 새 기능 폭증이 아니라 아래 세 가지입니다.

- 신뢰를 주는 설명
- 선택을 돕는 콘텐츠
- 운영 가능한 품질 정리

즉, 다음 단계는 "더 만들기"보다 "사용자가 이해하고 믿고 계속 쓰게 만들기" 쪽에 가깝습니다.
