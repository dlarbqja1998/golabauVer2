# 탈퇴 기능 기획서

## 배경

현재 서비스에는 로그아웃은 있지만 회원 탈퇴 기능이 없습니다.  
사용자는 직접 계정을 정리할 수 없고, 운영 문의로 우회해야 하며, 개인정보 처리에 대한 신뢰도도 떨어질 수 있습니다.

## 목표

- 마이페이지에서 사용자가 직접 탈퇴할 수 있어야 한다.
- 탈퇴 즉시 세션이 종료되어야 한다.
- 개인정보는 익명화 또는 삭제되어야 한다.
- 게시글/댓글은 서비스 기록 보호를 위해 익명 상태로 유지할 수 있어야 한다.

## 핵심 정책

### 1. 하드 삭제보다 소프트 삭제 우선

- 계정을 바로 지우지 않고 `DELETED` 상태로 전환한다.
- 공개 기록은 유지하되 작성자 표시는 `탈퇴한 사용자`로 바꾼다.

### 2. 즉시 삭제할 데이터

- 이메일
- 비밀번호
- 프로필 이미지
- 학과, 학년, 성별, 연락처
- 푸시 구독 정보
- 세션 쿠키

### 3. 탈퇴 제한 조건

- 진행 중인 모임의 방장인 경우
- 진행 중인 모임 신청 또는 매칭 내역이 있는 경우
- 관리자 계정인 경우

## 사용자 흐름

1. 마이페이지에서 `회원 탈퇴` 버튼을 누른다.
2. 삭제 범위와 유지 범위를 안내받는다.
3. 확인 문구를 입력한다.
4. 서버가 탈퇴 가능 여부를 검사한다.
5. 탈퇴 성공 시 로그아웃되고 홈으로 이동한다.

## 데이터 처리 방향

- `users.status = 'DELETED'`
- `users.deletedAt`, `users.anonymizedAt`, `users.deletionReason` 저장
- 이메일은 익명 이메일로 치환
- 닉네임은 `탈퇴한 사용자`로 고정
- `providerId`, 연락처, 프로필 정보는 `null`

## 오픈 이슈

- 탈퇴 사유 통계를 실제 운영에 활용할지 결정 필요
- 매칭 이력 보존 기간 정책 필요
- 운영자 화면에서 탈퇴 이력 조회 기능 추가 여부 검토 필요
