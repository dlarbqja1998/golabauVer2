# 탈퇴 기능 구현 계획

## 1. 목표

- 마이페이지에서 사용자가 직접 회원 탈퇴를 진행할 수 있게 한다.
- 탈퇴 계정은 즉시 세션이 종료되고 재로그인이 차단되어야 한다.
- 개인정보는 익명화하고, 커뮤니티 글/댓글은 `탈퇴한 사용자`로 남긴다.

## 2. 구현 범위

### 서버

- `users` 테이블에 탈퇴 관련 컬럼 추가
- 탈퇴 계정 공통 차단 로직 추가
- 마이페이지 탈퇴 액션 추가
- 푸시 구독 정리
- 카카오 로그인 콜백에서 탈퇴 계정 처리

### 클라이언트

- 마이페이지 위험 구역 UI 추가
- 탈퇴 안내 문구, 사유 선택, 확인 문구 입력 UI 추가
- 탈퇴 완료 시 홈 이동 처리
- 로그인 화면 오류 메시지 노출

### 문서

- `task.md` 기획서 유지
- 구현 후 `walkthrough.md`에 동작 방식 기록

## 3. 세부 작업

1. 스키마 확장
- `deletedAt`
- `anonymizedAt`
- `deletionReason`

2. 공통 인증 보강
- `DELETED` 상태 계정은 `getUserBySessionId`에서 제외
- 훅에서 탈퇴 계정 세션 쿠키 삭제

3. 탈퇴 액션 구현
- 확인 문구 검증
- 관리자 계정 탈퇴 제한
- 진행 중 모임/신청 상태 검사
- 푸시 구독 삭제
- 사용자 정보 익명화

4. UI 구현
- 마이페이지 하단 위험 구역 추가
- 탈퇴 사유 선택
- 확인 문구 입력
- 성공/실패 토스트 처리

5. 검증
- 타입 체크 실행
- 주요 시나리오 수동 점검

## 4. 위험 요소

- 모임 상태 테이블이 단순하지 않아 탈퇴 제한 조건을 잘못 잡을 수 있음
- 기존 문자열 인코딩이 섞여 있어 일부 화면 텍스트가 깨질 수 있음
- DB 마이그레이션이 실제 배포 DB에 반영되지 않으면 런타임 오류가 발생함

## 5. 완료 기준

- 마이페이지에서 탈퇴 시도가 가능하다.
- 진행 중 모임/신청이 있으면 탈퇴가 막힌다.
- 탈퇴 성공 시 세션이 삭제된다.
- 탈퇴 후 카카오 콜백으로 기존 계정이 복구되지 않는다.
